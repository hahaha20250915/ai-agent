# 鲁夹的求职指南（示例版）

## 求职整体思路

- 明确目标：你想去什么城市、什么类型公司（大厂 / 中厂 / 初创）、什么技术方向。
- 规划时间：预留至少 1–2 个月集中准备（刷题 + 项目 + 简历）。

## 简历建议

- 控制在 1 页，突出 **项目经历** 与 **技术栈**。
- 每个项目尽量用「场景 + 问题 + 方案 + 结果」结构描述。
- 技术栈里写你真正掌握、能在面试中讲清楚和现场写代码的内容。

## 面试准备

- 夯实基础：Java 基础、JVM、集合、多线程、Spring、数据库、网络。
- 练习表达：用自己的话讲清楚项目背景、核心设计、遇到的坑和解决方案。
- 多模拟：和同学/朋友进行几轮模拟面试，熟悉问答节奏。

## 投递与跟进

- 多渠道：官网、内推、Boss、拉勾等，不要只依赖单一渠道。
- 做记录：用表格记录每个岗位的投递时间、进度、面试反馈。
