# 鲁夹的项目学习建议（示例版）

## 如何选择练手项目

- **贴近实际业务**：包含接口调用、数据库、缓存、权限等。
- **可持续扩展**：后续可以不断加需求、做重构和优化。

## 学习路径

1. 先跑通：能启动项目、完成一个最小功能（例如简单的增删改查）。
2. 再理解：搞清楚每一层的职责（Controller / Service / Repository / 配置等）。
3. 再优化：日志、异常处理、参数校验、测试、重构。

## 实战建议

- 多画图：接口流程图、时序图、模块依赖图，有助于你自己和面试时讲解。
- 多写注释和文档：记录关键设计决策、踩过的坑以及解决方法。
